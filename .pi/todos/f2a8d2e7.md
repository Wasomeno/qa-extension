## Root Cause Found: Special Characters Dropped

### Problem
CDP `dispatchKeyEvent` was dropping special characters like `.` (period).

Expected: `admin@dot.co.id`
Actual: `admin@dotcoid` (missing `.` characters)

### Why
The `code` field was set to `Key.` for period, but it should be `Period`. Handling all special characters correctly with key events is complex.

### Solution
Use CDP's `Input.insertText` command instead of individual key events. This command:
- Is specifically designed for text input
- Handles all special characters correctly (`.`, `@`, `!`, etc.)
- Is more reliable and faster
- Is what Playwright/Puppeteer use internally

### Change Made
Replaced the character-by-character key event loop with a single `Input.insertText` call:

```typescript
await this.sendCommand(tabId, 'Input.insertText', {
  text: text,
});
```

### Test
User should reload extension and run test again. Should now type `admin@dot.co.id` correctly.
